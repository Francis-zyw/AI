# 版本记录与回滚约定

从 2026-03-26 起，本项目执行以下版本纪律：

## 1. 每次实质修改前

- 先记录 intake 状态
- 如果仓库还没有首个 commit，先建立基线提交
- 后续任何阶段性修改都必须形成可回滚 commit

## 2. 每次阶段性完成后

至少记录：

- 当前目标
- 修改范围
- 是否通过测试
- 是否影响运行结果
- git commit SHA

## 3. 推荐提交节奏

- 基线整理提交
- Step2 修复提交
- Step3 正式模式修复提交
- Review queue / 回写链路提交
- Step4 增量重跑与验收提交

## 4. 禁止事项

- 不把真实密钥写入仓库
- 不在大量修改后长期不提交
- 不在无法说明改动目的的情况下混乱提交

## 5. 回滚原则

- 优先通过 git commit 回滚
- 避免依赖聊天记录找回历史状态
- 每个关键里程碑都必须可通过 SHA 精确定位
